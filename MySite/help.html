User Documentation

 

1. Overview

The Quartz Professor Website Management System is a web based content management system which will allow professors of Boston Universityʼs Department of Computer Science to create websites that are uniform to a department standard. Professors will create a user account on the website managerʼs admin page. Once they have created an account professors will be able to input information on their classes, research, awards, etc. The website manager will then take this information and create a webpage that displays it in a professional and easily communicable way. All websites will follow the same style, to promote organization and departmental uniformity.

 

2. Creating a user account

·         A user starts by connecting to http://localhost/Quartz.

·         If the user does not already have an account on website manager they can click on the “create a new account” link at the bottom of the login box, and follow the rest of the steps. Otherwise the user may choose to just login in. (Please note that a professor is allowed only one account on the website manager).

·         After clicking on create a new account, the user should be redirected to a “create new account” page.

·         The user should enter the appropriate information in the following dialog boxes:

o   Name: the user should enter their first name

o   BU Email: the user should enter their BU email (note, this is the userʼs BU login name followed by @bu.edu)

o   Confirm email: the user should re-enter their same email for confirmation

o   Password: the user should enter a password for the site. Note this password must contain at least to special characters (non alphabet characters)

o   Confirm password: the user should re-enter the password they have selected for the site, for confirmation

o   BU ID: The user should enter their 9-digit BU ID. (Uxxxxxxxx)

·         After entering all of the appropriate information, and checking it for correctness, the user should hit submit, in the lower right hand corner.

·         After the admin has approved your account, an activation email will be sent when the admin approves the account

 

3. Creating a website

•      Return to the homepage either by clicking login in the upper right hand corner or by going to http://localhost/Quartz.

•      After creating an account, a user should log in with the username and password that they selected. Note: a user may not create a website until they have created an account.

•      After entering username and password in the appropriate fields a user should hit the login button. This will take the user to the “Website Management Panel.”

•      Once in the “Website Management Panel” a user may enter the following information and have it uploaded to the website:

o   Online/Offline: This will publish a userʼs website to the internet. A user should leave this option set to “offline” until they are ready for others to view their website.

o   My picture: A user may upload a picture of themselves, by clicking on the browse button, selecting the picture on their computer and then hitting the upload button. Uploaded photos will be automatically cropped but should not exceed 400KB in size. Note: it is suggested that professors use their official BU pictures.

o   General Information: The user should enter the following information in the appropriate fields.

1.      Display Name: The user should enter their name as they would like it to appear on their website

2.      Job Title: The user should enter their official BU job title.

3.      Office Address: The user should enter their office address, including their officeʼs room number

4.      Telephone: The user should enter their (office) telephone number

5.      Fax: The user should enter their officeʼs fax number

6.      Office Hours: The user should enter their office hours using the following format: [DAY HH:MM - HH:MM].

7.      At this point the user should hit the save, to make sure that the information entered in the above fields was not lost.

o   Short Biography: A user may upload a short bio about herself. There are no restrictions on content enforced by the system, thought it is strongly recommended that professors keep this biography professional. There is a fifteen-hundred character limit (1500 characters max). It is recommended that a user copy and paste into this field. (Note: The user should once again hit the save button located in this section to ensure that the information entered in the bio field was not lost.)

o   Teaching: The user should enter information about courses for which they are instructors:

1.      Course Name: The name of the course (as the user would like it to appear)

2.      Course Website Link: The URL to the courseʼs website

3.      Short Description: A brief description of the course.

o   After entering this information the user should select the add new button. If at any time the instructor wishes to remove courses, he / she can delete individual courses by clicking the “delete” button next to each or delete all by clicking “Delete all”

o   Research:

1.      Research Summary: The user can enter a synopsis (up to 1500 characters) about the research they are performing, or have performed in the past.

o   Awards:

1.      Enter a summary of your awards. There is no character limit. HTML formatting can be used.

2.      Check the enable box in order to display on website.

o   Projects:

1.      Enter a summary of your projects. There is no character limit. HTML formatting can be used.

2.      Check the enable box in order to display on website.

o   Students:

1.      Enter the names and information of your students. There is no character limit. HTML formatting can be used.

2.      Check the enable box in order to display on website.

o   Personal:

1.      Enter personal information or links. There is no character limit. HTML formatting can be used.

2.      Check the enable box in order to display on website.

 

Note: At this point the user should return to the top of the Website Management Panel and select the “online” option and press the apply button, to change the users website from being “under construction” to displaying all of the entered information.
